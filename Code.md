## Cell 5-7 — 尝试手势分割（失败的探索）

**目的**：重复文件里有多次手势，尝试自动分割成单个手势。

**方法**：
- 先算信号的 L2 范数（能量），看能不能找到"静止段"来切割
- 发现总能量基线很高（因为加速度计始终有 ~9.8 m/s² 的重力分量）
- 改用只看陀螺仪通道的能量，理论上静止时角速度为 0

**输出解读**：
```

```
陀螺仪能量范围 0-5 rad/s，均值 0.87。但即使只看陀螺仪，做手势的人在重复之间几乎不停顿，能量一直在 0.8-2.5 波动，找不到可靠的"静止区间"。

## Cell 8-10 — 分割函数 + 验证分割效果

**目的**：实现基于能量阈值的分割函数，并在所有手势上测试效果。

**算法**：平滑信号 → 阈值检测活跃区域 → 合并过近的段落。

**输出解读**：
```
wave: 6 segments from 5 files (expected ~26+ individual gestures)
```
5 个文件只分割出 6 段（平均每个文件 1-2 段），但预期应有 26+ 个单次手势。分割完全失败——它基本上把每个文件当成一整段。

**结论**：自动分割不可行，因为做手势的人是连续动作不停顿的。

## Cell 11-12 — Plan B：不分割，用 Cyclic LR-HMM

**关键设计决策**：放弃分割，直接用整个文件作为一条训练序列。

**为什么可行**：使用 **Left-to-Right Cyclic HMM**——允许从最后一个隐状态转移回第一个隐状态。这样模型可以自然地建模"手势重复"这个过程：做完一遍手势（走到最后一个状态）→ 立刻回到开头（回到第一个状态）→ 再做一遍。

**最终训练数据结构**：
```
每种手势: 6 条序列 = 5条重复文件(长) + 1条单次文件(短)
```

## Cell 13-15 — K-means 向量量化

**目的**：把连续的 6D IMU 向量离散化为整数标签，因为我们实现的是**离散 HMM**（观测值必须是有限集合中的整数）。

**方法**：把全部 83,451 个 6D 训练向量拼在一起 → 用 K-means 聚成 M=70 个簇 → 每个向量用最近簇的编号表示。

**为什么 M=70**：课程推荐 50-100 个簇，取中间偏上的值。太少则信息损失大、不同手势的观测重叠多；太多则每个簇样本太少，B 矩阵估计不稳。

**输出解读**：
```
cluster sizes: min=272, max=4899, mean=1192.2
empty clusters: 0
```
没有空簇，最小簇也有 272 个样本，说明 70 个簇的划分是合理的。柱状图也显示了簇大小分布——有几个大簇（常见姿态如静止）和很多小簇（特殊动作瞬间）。

## Cell 18-19 — HMM 类实现（核心代码）

**这是整个项目最核心的代码**，大约 110 行，实现了一个完整的 HMM。

### `__init__`：初始化

- **π（初始状态分布）**：Left-to-Right 结构下固定为 `[1, 0, ..., 0]`，表示序列一定从状态 0 开始
- **A（状态转移矩阵）**：每个状态只能停留在原地（自环）或前进到下一个状态。最后一个状态可以转回状态 0（cyclic）。初始时自环概率 ~0.5-0.7，前进概率是 1 减去自环
- **B（发射概率矩阵）**：随机初始化（Dirichlet 分布），加上 ε=1e-8 防止零概率
- **A_mask**：记录 A 矩阵的结构（哪些位置允许非零），训练时用来强制约束

### `forward(obs)` → `(alpha, c)`：前向算法

实现 Rabiner 论文的 Scaled Forward Algorithm（Section V）:
1. `α[:,0] = π * B[:,o₀]` — 初始：从初始分布出发，观测到第一个符号的概率
2. 每步 `α[:,t] = (A.T @ α[:,t-1]) * B[:,o_t]` — 递推：从前一步所有状态转移过来，乘以当前观测的发射概率
3. 每步用 scaling factor `c[t]` 归一化，防止数值下溢

`A.T @ α` 这个矩阵乘法的含义：对所有可能的来源状态 i，把 "在状态 i 的概率 × 从 i 转移到 j 的概率" 累加起来，得到到达状态 j 的总概率。

### `backward(obs, c)` → `beta`：后向算法

与 forward 方向相反，从序列末尾往回算：
1. `β[:,T-1] = 1/c[T-1]`
2. `β[:,t] = A @ (B[:,o_{t+1}] * β[:,t+1])`

和 forward 用相同的 scaling factor `c[t]`，保证两个方向的结果在数值上一致。

### `log_likelihood(obs)`：计算 log P(O|λ)

直接调用 forward，返回 `Σ log(c[t])`。这是 Rabiner scaling 的精髓——不需要在最后对 α 求和，log-likelihood 直接就是 scaling factor 的 log 之和。

### `train(obs_sequences, ...)` → `ll_history`：Baum-Welch (EM) 训练

这是参数学习的核心，Expectation-Maximization 算法：

**E-step（对每条训练序列）**：
1. 运行 forward + backward 得到 α, β
2. 计算 γ[i,t] = α[i,t]×β[i,t] / 归一化 — 表示"时刻 t 处于状态 i 的后验概率"
3. 计算 ξ[i,j,t] — 表示"时刻 t 从状态 i 转移到状态 j 的后验概率"
4. 把所有序列的统计量**累加**起来

**M-step**：
1. A_new[i,j] = Σξ[i,j,:] / Σγ[i,:] — 用统计量更新转移概率
2. B_new[j,k] = Σ_{t: o_t=k} γ[j,t] / Σγ[j,:] — 用统计量更新发射概率
3. 对 A 应用 mask 强制保持 Left-to-Right 结构
4. 对 B 加 ε=1e-8 防止零概率

**收敛判断**：LL 变化量 < tol 时停止。LL 必须单调不降，如果下降说明有 bug。

**内存优化**：ξ 的计算是逐步累加的（每个时刻算一个 N×N 矩阵加到 xi_sum 上），而不是存储完整的 (N, N, T-1) 数组。这在长序列上节省了大量内存。

## Cell 20-22 — Toy Data 验证

**目的**：在训练真实数据之前，先用一个简单的已知问题验证 HMM 实现的正确性。

**Toy Data 设计**：
- 3000 步的序列：前 1000 步在状态 0（只发射 obs=0），中间 1000 步在状态 1（随机发射 0 或 1），最后 1000 步回到状态 0
- 用 **ergodic** 拓扑（不是 Left-to-Right），因为状态会 0→1→0 回跳

**期望的学习结果**：
- A ≈ [[0.999, 0.001], [0.001, 0.999]]（状态很"粘"，转移概率很低）
- B ≈ [[1.0, 0.0], [0.5, 0.5]]（状态 0 只输出 0，状态 1 输出 0/1 各半）

**输出解读**：
```
converged at epoch 18, LL = -707.50
```
19 个 epoch 就收敛了。学到的参数和期望值高度一致，LL 曲线严格单调递增。**验证通过，HMM 实现正确。**

**补充说明**：`seed=4` 是试了几个种子选出的。`seed=42` 会陷入局部最优（两个状态学到几乎相同的 B 行），这验证了 EM 算法对初始化敏感、多次随机初始化的必要性。

## Cell 26 — Leave-One-Out 交叉验证 (LOOCV)

**目的**：评估模型的真实泛化能力。

**方法**：对每种手势的 6 条训练序列，轮流留出 1 条作为测试，用剩余 5 条重新训练该手势的 HMM，然后把留出的序列分别送入 6 个模型算 LL，看哪个模型给出最高 LL（即预测为哪种手势）。

**关键细节**：只有被测手势的 HMM 需要重新训练（因为它少了一条训练数据），其他 5 个手势的模型保持不变（它们的训练数据没变）。

**为什么不是简单验证**：如果直接用训练数据验证，存在**数据泄漏**——模型训练时见过这条数据，结果过于乐观。LOOCV 确保每条测试数据都不在训练集中。

**输出解读**：
```
Overall LOOCV: 35/36 = 97.2% Top-1, 36/36 = 100.0% Top-3
```

6 种手势 × 6 条序列 = 36 个测试样本：
- **35/36 正确** (97.2% Top-1)
- 唯一的错误：`beat3 [rep4]: pred=beat4 [WRONG]`
- 但 Top-3 里 beat3 排第二，所以 **Top-3 准确率 100%**

**beat3↔beat4 混淆分析**：beat3 是三拍节奏，beat4 是四拍节奏，动作模式最相似（都是上下拍打），是预期中最容易混淆的手势对。从 Top-3 列表也可以看到 beat3 和 beat4 经常互相出现在对方的 Top-2 里。

## 推理流程总结（test_classifier.py）

虽然不在 notebook 中，但课堂演示时可能需要说明推理是怎么跑的：

1. 加载 K-means 模型 + 6 个 HMM 模型
2. 读取测试文件 → 丢掉时间戳列 → 得到 6D IMU 数据
3. K-means predict → 把 6D 向量序列转成整数标签序列
4. 对 6 个 HMM 分别跑 forward 算法 → 得到 6 个 log-likelihood
5. LL 最高的模型对应的手势就是预测结果
6. 输出 Top-3 预测及对应 LL 值
